### Crackme0x01

Let's analyze the second crackme using a more automated approach. We'll start by launching radare2 with automatic analysis enabled with `r2 -AA crackme0x01` :

```bash
[0x080483e4]> pdd
/* r2dec pseudo code output (r2 5.9.9) */
/* crackme0x01 @ 0x80483e4 */
#include <stdint.h>

int32_t main_80483e4 (void) {
    uint32_t var_4h;
    int32_t var_sp_4h;
    eax = 0;
    eax += 0xf;
    eax += 0xf;
    eax >>= 4;
    eax <<= 4;
    printf ("IOLI Crackme Level 0x01\n");
    printf ("Password:");
    eax = &var_4h;
    scanf (sub.Invalid_Password__n:_804854c, eax);
    if (var_4h != 0x149a) {
        printf ("Invalid Password!\n");
    } else {
        printf ("Password OK :)\n");
    }
    eax = 0;
    return eax;
}
```

The decompiled code reveals a crucial comparison: our input (`var_4h`) is checked against the hexadecimal value `0x149a`. Let's convert this value to different bases to find the correct password:

```bash
[0x080483e4]> ? 0x149a
```

The decimal value `5274` is our password! This value, when converted to hexadecimal, matches the comparison value `0x149a` in the code.

### Crackme0x02

Let's analyze the third crackme using radare2 with automatic analysis enabled using `r2 -AA crackme0x02`. After loading the binary, we can examine the main function to understand the password validation logic:

```bash
...
    scanf (0x804856c, eax);
    var_8h = 0x5a;
    var_ch = 0x1ec;
    edx = 0x1ec;
    eax = &var_8h;
    *(eax) += edx; // var_8h = 0x5a + 0x1ec warning & and *
    eax = var_8h; 
    eax *= var_8h; // eax * eax = var_8h * var_8h = eax * var_8h
    var_ch = eax;
    eax = var_4h;
    if (eax == var_ch) { // var_ch = (0x5a+0x1ec)^2
        printf ("Password OK :)\n");
    } else {
        printf ("Invalid Password!\n");
    }
...
```
By analyzing the code, we can see that the password validation involves several mathematical operations:
1. `var_8h` is initialized to 0x5a (90)
2. `var_ch` is initialized to 0x1ec (492)
3. These values are added together: 90 + 492 = 582
4. The result is squared: 582 * 582 = 338724

Therefore, the correct password is `338724`.

> Bonus: We can also find the password using debugging. After disassembling main with `pdf@main`, we can place a breakpoint after the multiplication instruction with `db 0x08048448`, run the program with `dc`, and then print the value of eax with `? eax` to get 338724.
